# Skill Evaluations

This directory contains lightweight evaluations for agile-development-skills.

The goal is to verify whether an AI coding agent actually follows the intended workflow.

Examples:

- TDD: verify test-first behavior
- Refactoring: verify tests still pass
- Legacy code: verify characterization tests are added first
- Feedback loop: verify diagnostics and tests are re-run after edits
- Code review: verify seeded defects are detected

Future direction:

- repository-based benchmark tasks
- automated scoring
- diff-based verification
- regression tracking across models and agents
